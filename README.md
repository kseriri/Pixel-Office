<h1 align="center">Pixel Office</h1>

<p align="center">
  <em>あなたのAIコーディングエージェントを、ドット絵オフィスで働く社員として眺めるツール。</em>
</p>

Pixel Office は、いま自分のマシンで動いている AI コーディングエージェント（Claude Code や
Codex）を、ドット絵のスタートアップ風オフィスの「社員キャラクター」として可視化するツールです。
プロジェクトごとに部屋が分かれ、各エージェントは自分のデスクで作業し、頭上のバッジで使用中の
AIモデルが分かり、ターンが終わると休憩室へ移動します。

エージェントが残す**ローカルのセッションログを監視して**動きをリアルタイムに描画するだけで、
**外部にデータを送信しません**（すべて手元で完結します）。

> Pixel Office は、オープンソースの
> [**Pixel Agents**](https://github.com/pixel-agents-hq/pixel-agents)（作者: Pablo De Lucca、
> MITライセンス）をフォークして独自にカスタマイズしたものです。詳細は
> [クレジット](#クレジット) を参照してください。

## できること

- **プロジェクトごとの部屋** — 2×2 の作業部屋（各プロジェクト専用）＋共有の休憩室。同じ
  プロジェクトのエージェントは同じ部屋に集まります。
- **休憩室への移動** — エージェントはターンが終わると休憩室のソファへ歩いていき、また作業を
  始めるとデスクに戻ります。
- **モデルバッジ** — 各キャラの頭上に、使用モデル（Opus / Sonnet / Haiku / GPT / Codex …）と
  累積トークン数・概算コスト（キャッシュ考慮）を表示。`fable` モデルだけは光輪と羽が付きます ✨
- **モニタールーム** — 休憩室の下半分はオペレーションルームになっていて、大画面に
  **プロジェクト別トークン使用量グラフ**（モデル色分け）がリアルタイム表示されます。
- **複数ツール対応** — **Claude Code**（`~/.claude`）と **Codex**（`~/.codex`）のセッションを
  同じオフィスに並べて表示します。
- **にぎやかなオフィス** — 観葉植物、壁アート、ホワイトボード、壁掛けの部屋名看板、歩き回る
  ペットなど。
- **どこでも動く** — 小さなローカルサーバがオフィス画面をブラウザに配信。PWA としてインストール
  すれば独立したアプリウィンドウとして使えます（Electron のような重いダウンロードは不要）。

## 仕組み（データはどこから来るか）

Pixel Office 本体は「ローカルサーバ」と「ブラウザ画面」の2つで構成されます。

```
AIエージェント（Claude Code / Codex）
   └─ セッションログを書き出す
        ~/.claude/projects/... , ~/.codex/sessions/...
              │  （ローカルサーバが監視・解析）
              ▼
   ローカルサーバ（dist/cli.js, 127.0.0.1 のみで待受）
              │  WebSocket
              ▼
   ブラウザ画面（ドット絵オフィス）
```

- サーバは **127.0.0.1（localhost）でのみ待ち受け**、外部公開しません。
- ログを読み取って解析するだけで、内容をどこにも送りません。

## 動かし方

```bash
# 依存関係のインストール
npm install

# ビルド（サーバ dist/cli.js と Web UI dist/webview を生成）
npm run build

# 起動（ポートは任意）
node dist/cli.js --port 3100
```

ブラウザで `http://127.0.0.1:3100` を開くとオフィスが表示されます。

起動方法は3通りあります。用途で選んでください：
1. **ブラウザ**（上記・一番手軽）
2. **PWAとしてインストール**（アプリ窓・アイコン付き。下記）
3. **デスクトップアプリ（Electron）**（ネイティブ窓・サーバ同梱で自動起動。下記）

### 自分のセッションを表示させる

いつも通りエージェントを起動するだけです。Pixel Office が `~/.claude/projects` と
`~/.codex/sessions` のセッションログを監視し、稼働中のセッションを自動でキャラクター化します。

> Claude の場合、アプリ自身の作業フォルダ**以外**で起動したセッションも表示するには、
> `~/.pixel-agents/config.json` の `watchAllSessions` を `true` にする必要があります
> （[設定ファイル](#設定ファイル) 参照）。

## アプリとしてインストール（PWA）

`http://127.0.0.1:3100` を Chrome / Edge で開き、アドレスバーの「インストール」アイコン
（または ⋮ メニュー →「Pixel Office をインストール」）を押すと、**アイコン付きの独立ウィンドウ**
として起動できるようになります。ブラウザのタブ枠が消え、アプリのように使えます。

---

## デスクトップアプリ（Electron）

ブラウザを使わずネイティブウィンドウで起動したい場合は、同梱の Electron シェルを使います。
サーバ（`dist/cli.js`）をウィンドウが自動で立ち上げるので、`node dist/cli.js` を別途実行する
必要はありません。

```bash
npm run build            # 先にサーバ + UI をビルド（未ビルドなら）
cd electron
npm install              # Electron を取得（初回のみ・約200MBと重い）
npm start                # ネイティブウィンドウで起動
```

Electron はサイズが大きいので、この `electron/` フォルダは**任意インストール**です（ルートの
`npm install` には含めていません）。ブラウザ/PWA だけで使う人は入れなくて構いません。
配布用のインストーラ（.dmg / .exe）を作るには electron-builder 等の追加設定が必要です。

## オフィスのレイアウトをカスタムする

オフィスのレイアウト（部屋・色・家具の配置）は Python スクリプトで生成しています。
スクリプトを編集して再生成する方式なので、手でタイルを並べる必要はありません。

```bash
# scripts/gen-rooms-layout.py を編集（部屋・色・家具）してから：
python3 scripts/gen-rooms-layout.py webview-ui/public/assets/default-layout-N.json
npm run build
```

- `N`（レイアウトのリビジョン番号）を1つ上げると、既存のインストールでも新レイアウトに
  自動で切り替わります。
- スクリプトは実行時に **ASCIIプレビュー**を出力するので、ビルド前に配置を確認できます。

## 設定ファイル

`~/.pixel-agents/config.json` に設定が保存されます（`standalone` セクションが本アプリ用）。

| キー | 説明 |
|---|---|
| `watchAllSessions` | `true` で、他フォルダ／他プロジェクトの稼働セッションも表示（既定 `false`） |
| `soundEnabled` | 通知音のオン/オフ |
| `alwaysShowLabels` | キャラのラベルを常に表示 |

レイアウトは `~/.pixel-agents/layout.json` に保存されます。バンドルされた既定レイアウトの
リビジョンが新しくなると、古いユーザーレイアウトは自動でリセットされます。

## 困ったとき

- **キャラが出てこない** → 該当セッションが稼働中か、`watchAllSessions` が `true` かを確認。
  アプリを多重起動していると同じポートを取り合うので、1つだけ起動してください。
- **モデルバッジ／トークンが出ない** → そのセッションが最初のアシスタント応答を出した後
  （ログにモデル情報が現れてから）表示されます。
- **Codex のキャラが出ない** → 直近に稼働した Codex セッション（`~/.codex/sessions`）が必要です。
